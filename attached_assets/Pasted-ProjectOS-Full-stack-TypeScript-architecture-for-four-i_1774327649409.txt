ProjectOS: Full-stack TypeScript architecture for four interconnected systems
The optimal architecture for ProjectOS combines BullMQ for reliable reminder scheduling, SendGrid Inbound Parse and AWS SES/SNS for dual-path email reception, the googleapis SDK with encrypted token storage for Gmail integration, and Drizzle ORM's pgTable API for a type-safe PostgreSQL schema. This plan lays out every file, schema definition, and code module needed to ship these four systems on a Replit-hosted Express + React + PostgreSQL stack. Each system is isolated into its own Express router module, shares a unified Drizzle schema, and communicates through well-defined internal service functions.

AES-256-GCM encryption protects stored OAuth tokens
typescript
// src/server/lib/crypto.ts
import { createCipheriv, createDecipheriv, randomBytes } from 'crypto';
import { env } from '../env';

const ALGORITHM = 'aes-256-gcm';
const KEY = Buffer.from(env.TOKEN_ENCRYPTION_KEY, 'hex'); // 32 bytes

export function encrypt(plaintext: string): string {
  const iv = randomBytes(16);
  const cipher = createCipheriv(ALGORITHM, KEY, iv);
  const encrypted = Buffer.concat([cipher.update(plaintext, 'utf8'), cipher.final()]);
  const tag = cipher.getAuthTag();
  // Format: iv:tag:ciphertext (all hex-encoded)
  return `${iv.toString('hex')}:${tag.toString('hex')}:${encrypted.toString('hex')}`;
}

export function decrypt(encoded: string): string {
  const [ivHex, tagHex, ciphertextHex] = encoded.split(':');
  const iv = Buffer.from(ivHex, 'hex');
  const tag = Buffer.from(tagHex, 'hex');
  const ciphertext = Buffer.from(ciphertextHex, 'hex');
  const decipher = createDecipheriv(ALGORITHM, KEY, iv);
  decipher.setAuthTag(tag);
  return Buffer.concat([decipher.update(ciphertext), decipher.final()]).toString('utf8');
}
The GCM mode provides both confidentiality and authentication — any tampered ciphertext will throw on decryption. 
Gitbooks
 The iv is unique per encryption and stored alongside the ciphertext, while the single TOKEN_ENCRYPTION_KEY is a 256-bit hex string generated once via openssl rand -hex 32.

System 1: Reminder scheduling with BullMQ and multi-channel dispatch
The reminder system uses BullMQ delayed jobs as its primary mechanism. When a user creates a reminder, the backend inserts a DB row and enqueues a BullMQ job with a delay equal to scheduledAt - now. A separate worker process picks up fired jobs and dispatches through the correct channel. A node-cron sweep runs every minute as a safety net to catch any jobs that Redis might have lost.

export async function makeVoiceCall(to: string, message: string): Promise<string> {
  const twiml = `<Response><Say voice="alice" language="en-US">${escapeXml(message)}</Say></Response>`;
  const call = await client.calls.create({
    to,
    from: env.TWILIO_PHONE_NUMBER,
    twiml,
  });
  return call.sid;
}

function escapeXml(str: string): string {
  return str.replace(/&/g, '&amp;').replace(/</g, '&lt;')
    .replace(/>/g, '&gt;').replace(/"/g, '&quot;');
}

// src/server/jobs/queue.ts
import { Queue } from 'bullmq';
import IORedis from 'ioredis';
import { env } from '../env';

export const connection = new IORedis(env.REDIS_URL, { maxRetriesPerRequest: null });

export interface ReminderJobData {
  reminderId: number;
  userId: number;
  notificationType: 'sms' | 'call' | 'email' | 'in_app';
  target: string;
  title: string;
  message: string;
}

export const reminderQueue = new Queue<ReminderJobData>('reminders', { connection });

export async function scheduleReminderJob(
  data: ReminderJobData,
  scheduledAt: Date
): Promise<string> {
  const delay = Math.max(scheduledAt.getTime() - Date.now(), 0);
  const job = await reminderQueue.add('send-reminder', data, {
    delay,
    attempts: 3,
    backoff: { type: 'exponential', delay: 5000 },
    removeOnComplete: { count: 1000 },
    removeOnFail: { count: 5000 },
  });
  return job.id!;
}

// src/server/jobs/reminderWorker.ts
import { Worker, Job } from 'bullmq';
import { connection, ReminderJobData } from './queue';
import { sendSMS, makeVoiceCall } from '../services/twilio.service';
import { sendEmailViaSendGrid } from '../services/sendgrid.service';
import { createInAppNotification } from '../services/notification.service';
import { db } from '../db';
import { reminders } from '../db/schema';
import { eq } from 'drizzle-orm';

const worker = new Worker<ReminderJobData>(
  'reminders',
  async (job: Job<ReminderJobData>) => {
    const { reminderId, notificationType, target, title, message } = job.data;
    try {
      switch (notificationType) {
        case 'sms':
          await sendSMS(target, `${title}: ${message}`);
          break;
        case 'call':
          await makeVoiceCall(target, `${title}. ${message}`);
          break;
        case 'email':
          await sendEmailViaSendGrid(target, title, message);
          break;
        case 'in_app':
          await createInAppNotification(job.data.userId, title, message);
          break;
      }
      await db.update(reminders)
        .set({ status: 'sent', sentAt: new Date() })
        .where(eq(reminders.id, reminderId));
    } catch (error: any) {
      await db.update(reminders)
        .set({ status: 'failed', errorMessage: error.message })
        .where(eq(reminders.id, reminderId));
      throw error; // BullMQ will retry based on attempts config
    }
  },
  { connection, concurrency: 10 }
);

worker.on('failed', (job, err) => {
  console.error(`Reminder job ${job?.id} failed: ${err.message}`);
});

export { worker };

cron.schedule('* * * * *', async () => {
  const dueReminders = await db.select().from(reminders)

// src/server/middleware/verifySendgrid.ts
import { Request, Response, NextFunction } from 'express';
import { EventWebhook, EventWebhookHeader } from '@sendgrid/eventwebhook';
import { env } from '../env';

export function verifySendgridSignature(req: Request, res: Response, next: NextFunction) {
  if (!env.SENDGRID_WEBHOOK_PUBLIC_KEY) return next();
  const signature = req.headers[EventWebhookHeader.SIGNATURE().toLowerCase()] as string;
  const timestamp = req.headers[EventWebhookHeader.TIMESTAMP().toLowerCase()] as string;
  const ew = new EventWebhook();
  const key = ew.convertPublicKeyToECDSA(env.SENDGRID_WEBHOOK_PUBLIC_KEY);
  if (!ew.verifySignature(key, (req as any).rawBody, signature, timestamp)) {
    return res.status(401).json({ error: 'Invalid webhook signature' });
  }
  next();
}

// src/server/middleware/verifyTwilio.ts
import twilio from 'twilio';
import { Request, Response, NextFunction } from 'express';
import { env } from '../env';

export function verifyTwilioSignature(req: Request, res: Response, next: NextFunction) {
  const signature = req.headers['x-twilio-signature'] as string;
  const protocol = req.headers['x-forwarded-proto'] || req.protocol;
  const url = `${protocol}://${req.get('host')}${req.originalUrl}`;
  if (!twilio.validateRequest(env.TWILIO_AUTH_TOKEN, signature, url, req.body)) {
    return res.status(403).json({ error: 'Invalid Twilio signature' });
  }
  next();
}

// src/server/services/emailParser.service.ts
import { db } from '../db';
import { emailRoutes, projects, emailLogs } from '../db/schema';
import { eq } from 'drizzle-orm';

/** Extract [ProjectTag] from subject line */
export function extractProjectTag(subject: string): string | null {
  const match = subject.match(/\[([^\]]+)\]/);
  return match ? match[1] : null;
}

/** Match email to project by address or subject tag */
export async function matchProject(
  toAddress: string,
  subject: string
): Promise<number | null> {
  // Strategy 1: unique address mapping
  const [route] = await db.select()
    .from(emailRoutes)
    .where(eq(emailRoutes.assignedEmail, toAddress.toLowerCase()));
  if (route) return route.projectId;

  // Strategy 2: subject line tag
  const tag = extractProjectTag(subject);
  if (tag) {
    const [project] = await db.select()
      .from(projects)
      .where(eq(projects.tag, tag));
    if (project) return project.id;
  }
  return null;
}

/** Store parsed email in logs */
export async function storeInboundEmail(params: {
  projectId: number | null;
  fromAddress: string;
  toAddress: string;
  subject: string;
  bodyText: string | null;
  bodyHtml: string | null;
  provider: 'sendgrid' | 'ses' | 'gmail';
  attachments?: Array<{ filename: string; contentType: string; size: number }>;
}) {
  return db.insert(emailLogs).values(params).returning();
}

// POST /api/webhooks/ses-inbound — AWS SES via SNS
router.post('/webhooks/ses-inbound', asyncHandler(async (req, res) => {
  const snsMessage = req.body;

  // Handle SNS subscription confirmation
  if (snsMessage.Type === 'SubscriptionConfirmation') {
    await fetch(snsMessage.SubscribeURL);
    return res.status(200).send('Confirmed');
  }

  if (snsMessage.Type !== 'Notification') {
    return res.status(200).send('OK');
  }

  const notification = JSON.parse(snsMessage.Message);
  const mail = notification.mail;
  const commonHeaders = mail.commonHeaders;

  // If SES stored in S3, fetch and parse the raw MIME
  let bodyText: string | null = null;
  let bodyHtml: string | null = null;

  if (notification.receipt?.action?.type === 'S3') {
    const s3 = new S3Client({ region: env.AWS_SES_REGION });
    const obj = await s3.send(new GetObjectCommand({
      Bucket: notification.receipt.action.bucketName,
      Key: notification.receipt.action.objectKey || mail.messageId,
    }));
    const rawEmail = await obj.Body!.transformToString();
    const parsed = await simpleParser(rawEmail);
    bodyText = parsed.text || null;
    bodyHtml = parsed.html || null;
  } else if (notification.content) {
    // SNS action includes content directly (max 150KB)
    const parsed = await simpleParser(notification.content);
    bodyText = parsed.text || null;
    bodyHtml = parsed.html || null;
  }

  const toAddress = commonHeaders.to?.[0] || mail.destination?.[0] || '';
  const subject = commonHeaders.subject || '';
  const projectId = await matchProject(toAddress, subject);

  await storeInboundEmail({
    projectId,
    fromAddress: commonHeaders.from?.[0] || mail.source,
    toAddress,
    subject,
    bodyText,
    bodyHtml,
    provider: 'ses',
  });
  res.status(200).send('OK');
}));

export function createOAuth2Client() {

    env.GOOGLE_CLIENT_SECRET,

  );

export function getAuthUrl(): string {
  const oauth2Client = createOAuth2Client();
  return oauth2Client.generateAuthUrl({

    prompt: 'consent',  // forces refresh_token on every authorization
    scope: SCOPES,
  });

/** Exchange auth code for tokens, encrypt, store in DB */
export async function handleOAuthCallback(code: string, userId: number) {
  const oauth2Client = createOAuth2Client();
  const { tokens } = await oauth2Client.getToken(code);

  oauth2Client.setCredentials(tokens);
  const gmail = google.gmail({ version: 'v1', auth: oauth2Client });
  const profile = await gmail.users.getProfile({ userId: 'me' });
  const email = profile.data.emailAddress!;

  await db.insert(gmailAccounts).values({
    userId,
    email,
    accessTokenEncrypted: encrypt(tokens.access_token!),
    refreshTokenEncrypted: encrypt(tokens.refresh_token!),
    tokenExpiryDate: tokens.expiry_date ? new Date(tokens.expiry_date) : null,
    scopes: SCOPES.join(' '),
  }).onConflictDoUpdate({
    target: gmailAccounts.email,
    set: {
      accessTokenEncrypted: encrypt(tokens.access_token!),
      refreshTokenEncrypted: encrypt(tokens.refresh_token!),
      tokenExpiryDate: tokens.expiry_date ? new Date(tokens.expiry_date) : null,
    },
  });

  return { email };
}

/** Restore an authenticated Gmail client from stored tokens */
export async function getAuthedGmailClient(accountId: number) {
  const [account] = await db.select().from(gmailAccounts)
    .where(eq(gmailAccounts.id, accountId));
  if (!account) throw new Error('Gmail account not found');

  const oauth2Client = createOAuth2Client();
  oauth2Client.setCredentials({
    access_token: decrypt(account.accessTokenEncrypted),
    refresh_token: decrypt(account.refreshTokenEncrypted),
    expiry_date: account.tokenExpiryDate?.getTime(),
  });

  // Listen for automatic token refresh events and persist new tokens
  oauth2Client.on('tokens', async (newTokens) => {
    const updates: Record<string, any> = {
      accessTokenEncrypted: encrypt(newTokens.access_token!),
    };
    if (newTokens.refresh_token) {
      updates.refreshTokenEncrypted = encrypt(newTokens.refresh_token);
    }
    if (newTokens.expiry_date) {
      updates.tokenExpiryDate = new Date(newTokens.expiry_date);
    }
    await db.update(gmailAccounts).set(updates).where(eq(gmailAccounts.id, accountId));
  });

  return google.gmail({ version: 'v1', auth: oauth2Client });
}

/** Decode Gmail base64url-encoded body part */
function decodeBody(data: string): string {
  return Buffer.from(data, 'base64url').toString('utf8');
}

/** Recursively find body parts in multipart messages */
function findBodyParts(
  payload: any
): { text: string | null; html: string | null } {
  let text: string | null = null;
  let html: string | null = null;

  if (payload.mimeType === 'text/plain' && payload.body?.data) {
    text = decodeBody(payload.body.data);
  } else if (payload.mimeType === 'text/html' && payload.body?.data) {
    html = decodeBody(payload.body.data);
  }

  if (payload.parts) {
    for (const part of payload.parts) {
      const nested = findBodyParts(part);
      if (nested.text) text = nested.text;
      if (nested.html) html = nested.html;
    }
  }
  return { text, html };
}

/** Fetch and parse messages from Gmail inbox */
export async function syncInbox(accountId: number, maxResults = 50) {
  const gmail = await getAuthedGmailClient(accountId);
  const [account] = await db.select().from(gmailAccounts)
    .where(eq(gmailAccounts.id, accountId));

  const listResponse = await gmail.users.messages.list({
    userId: 'me',
    maxResults,
    q: 'in:inbox',
  });

  const messageIds = listResponse.data.messages || [];
  const results = [];

  // Process in chunks of 20 to respect rate limits
  for (let i = 0; i < messageIds.length; i += 20) {
    const chunk = messageIds.slice(i, i + 20);
    const messages = await Promise.all(
      chunk.map(m => gmail.users.messages.get({
        userId: 'me', id: m.id!, format: 'full',
      }))
    );

    for (const msg of messages) {
      const headers = msg.data.payload?.headers || [];
      const getHeader = (name: string) =>
        headers.find(h => h.name?.toLowerCase() === name.toLowerCase())?.value || '';

      const subject = getHeader('Subject');
      const from = getHeader('From');
      const date = getHeader('Date');
      const { text, html } = findBodyParts(msg.data.payload);

      results.push({
        gmailMessageId: msg.data.id!,
        subject, from, date,
        bodyText: text, bodyHtml: html,
      });
    }
  }
  return results;
}
Key implementation details: the prompt: 'consent' parameter in generateAuthUrl forces Google to return a fresh refresh_token every time, solving the common gotcha where refresh tokens only appear on first authorization. The oauth2Client.on('tokens') listener automatically persists refreshed access tokens to the database, so the system is self-healing when tokens expire. Body extraction handles Gmail's recursive multipart structure via a depth-first walk of the parts tree, decoding each base64url segment with Node's native Buffer.from(data, 'base64url').

WebSocket server enables real-time in-app notifications

// src/server/websocket.ts
import { Server as HttpServer } from 'http';
import { WebSocketServer, WebSocket } from 'ws';

const clients = new Map<number, Set<WebSocket>>(); // userId -> connections

export function initWebSocket(server: HttpServer) {
  const wss = new WebSocketServer({ server, path: '/ws/notifications' });

  wss.on('connection', (ws, req) => {
    // Expect userId as query param (in production, verify JWT here)
    const url = new URL(req.url!, `http://${req.headers.host}`);
    const userId = parseInt(url.searchParams.get('userId') || '0');
    if (!userId) { ws.close(); return; }

    if (!clients.has(userId)) clients.set(userId, new Set());
    clients.get(userId)!.add(ws);

    ws.on('close', () => {
      clients.get(userId)?.delete(ws);
      if (clients.get(userId)?.size === 0) clients.delete(userId);
    });
  });
}

export function broadcastToUser(userId: number, payload: object) {
  const sockets = clients.get(userId);
  if (!sockets) return;
  const message = JSON.stringify(payload);
  for (const ws of sockets) {
    if (ws.readyState === WebSocket.OPEN) ws.send(message);
  }
}

Key architectural decisions and their rationale
BullMQ over pure node-cron for reminders. node-cron is memory-only — a process restart loses all scheduled jobs. BullMQ persists delayed jobs in Redis, supports automatic retries with exponential backoff, and provides changeDelay() for rescheduling. The node-cron sweep is retained only as a fallback that catches any reminder where BullMQ's delay somehow didn't fire, giving the system belt-and-suspenders reliability.

Dual email receiving paths are necessary for resilience. SendGrid Inbound Parse requires only MX record configuration and sends clean multipart form data. AWS SES via SNS requires more setup (receipt rules, SNS topics, optional S3 buckets) but is dramatically cheaper at scale ($0.10 per 1,000 emails vs SendGrid's metered pricing) and provides spam/virus/SPF/DKIM verdicts in the receipt metadata. Running both paths means you can fail over between providers.

Gmail token encryption uses AES-256-GCM specifically because GCM mode provides authenticated encryption — any bit-flip or truncation of stored ciphertext will cause decryption to fail with an authentication error rather than silently producing garbage. The IV is regenerated per encryption and stored alongside the ciphertext, so identical plaintext tokens produce different ciphertext. The single hex-encoded TOKEN_ENCRYPTION_KEY avoids the cost explosion of per-token KMS calls.

Drizzle's $type<T>() generic on jsonb columns bridges PostgreSQL's schemaless JSON storage with TypeScript's type system. The attachments column typed as $type<Array<{filename: string; contentType: string; size: number}>>() means every query result is fully typed at compile time, while the database stores flexible JSON. The $inferSelect and $inferInsert type helpers eliminate manual type duplication — insert types correctly make defaultNow() columns optional while select types include them as required.

SendGrid's Inbound Parse webhook sends multipart/form-data, not JSON. This is the most common mistake in implementation — using express.json() on the webhook endpoint will silently fail. The route uses multer middleware with no storage configuration (memory mode) to parse the multipart payload. Fields arrive as req.body.from, req.body.subject, req.body.text, req.body.html, while file attachments appear in req.files.

The Gmail oauth2Client.on('tokens') event listener is the critical piece that makes the system self-healing. When the googleapis library automatically refreshes an expired access token (which happens transparently 5 minutes before expiry), this listener fires and the new token is encrypted and persisted to PostgreSQL. Without this listener, refreshed tokens exist only in memory and are lost on restart, forcing users to re-authenticate.

Conclusion
This architecture separates concerns cleanly: each of the four systems is a standalone Express router module, all sharing the unified Drizzle schema and service layer. The BullMQ + node-cron hybrid gives you both precision scheduling and crash resilience for reminders. The dual-path webhook design (SendGrid + SES) gives flexibility and provider redundancy for inbound email. Gmail OAuth tokens are encrypted at rest with AES-256-GCM and automatically refreshed via the tokens event. The WebSocket layer enables real-time in-app notification delivery alongside the DB-backed polling fallback. Every module uses strict TypeScript, Zod validation on inputs, and the asyncHandler wrapper to ensure errors flow through Express's error middleware rather than crashing the process.
